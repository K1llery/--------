# 优途前端品牌升级与界面改版设计

日期：2026-05-14

## 背景

当前前端已经具备目的地推荐、地图导航、设施查询、旅游规划、旅游日记等核心功能，但整体视觉仍偏课程项目模板：品牌识别不稳定、首页产品表达不够清晰、全局控件和色彩体系缺乏统一。现有品牌入口仍使用“城市漫游”，与项目后续对外展示名称“优途”不一致。

本次改版不调整信息架构、不改动后端接口、不新增复杂交互流程，目标是在保留现有页面和功能的前提下，完成一次稳妥但明显的产品化升级。

## 目标

1. 全站品牌统一为“优途”，替换旧品牌文案与品牌入口表现。
2. 基于现有品牌资源 `优途.png` 建立统一的视觉系统，形成清爽专业的旅游产品气质。
3. 重做全局应用壳子、首页、按钮、卡片、表单和状态块的视觉语言，让现有功能页自然受益。
4. 保持现有路由、页面结构、数据接口和主要交互不变，控制改版风险。

## 非目标

- 不调整现有页面路由与信息架构。
- 不改动后端接口、数据模型或业务逻辑。
- 不重写所有功能页，仅通过全局样式和关键入口页面带动整体观感升级。
- 不引入外部品牌系统、重型动画库或复杂主题切换机制。

## 品牌与视觉方向

### 品牌定位

“优途”应体现如下感受：

- 路线可信
- 城市探索
- 轻出行服务
- 清爽专业

对应用户第一印象应是：这是一个可用于目的地浏览、路线规划和旅游组织的完整产品，而不是单页展示或课堂原型。

### logo 资产

- 使用仓库现有文件 `优途.png` 作为品牌视觉来源。
- logo 中的定位点、山路、日出构成品牌识别基础，页面配色和局部图形语言应围绕这三类元素展开。
- 不保留当前 `App.vue` 中的通用地球图标作为主品牌标识。

### 色彩策略

品牌主色不再使用现有偏紫体系，改为蓝橙体系：

- 主色：地图蓝，用于导航激活态、主按钮、链接强调、关键数字
- 强调色：日出橙，用于主要 CTA、状态强调、局部高亮
- 背景色：浅雾蓝与白色组合，保证地图、卡片、表单可读性
- 中性色：冷灰系，用于正文、说明文案、边框、分隔和禁用态

原则：

- 蓝色承担“可信与结构”
- 橙色承担“行动与注意力”
- 避免重新落回紫色主导的视觉中心
- 控制高饱和颜色面积，避免首页像营销海报

### 字体与层级

- 保留当前中文友好的系统字体栈，不新增高风险网络字体依赖
- 通过字号、字重、间距和留白重建层级，不依赖装饰性字体
- 首页标题、区块标题、卡片数字、辅助文案要形成稳定层次

### 质感原则

- 少做漂浮装饰和泛渐变背景，更多使用干净分区、稳定阴影和清楚边界
- 卡片圆角与阴影统一收敛，减少“拼贴感”
- 所有交互元素具有明确 hover、focus、active 状态
- 动效以轻量过渡为主，不引入答辩时容易失控的花哨动画

## 改造范围

### 1. 全局应用壳子

核心文件：`frontend/src/App.vue`

目标：

- 桌面侧边栏与移动端头部统一为“优途”品牌入口
- 用 logo 图片替换当前通用图标品牌块
- 统一导航、顶部工具条、账号区域的视觉节奏

设计要求：

- 品牌区应包含 logo、品牌名“优途”和一句简短辅助描述
- 导航链接强化当前选中态，未选中态更克制
- 顶部工具按钮从“普通文字按钮”升级为更统一的产品工具栏样式
- 登录/注册/收藏/通知/消息的层级和密度要更接近真实产品后台，而不是课程页面按钮堆叠

### 2. 首页重做

核心文件：`frontend/src/pages/HomePage.vue`

首页不再只是“有几个模块的内容集合”，而要成为产品首页。

#### Hero 区

必须完成：

- 第一屏明确展示品牌名“优途”
- 用一句短标题表达产品核心价值，例如围绕“城市探索 + 路线规划 + 个性化出行”
- 保留两个核心主动作：浏览目的地、开始导航
- Hero 媒体区域继续使用真实目的地图片，但整体构图要更像产品入口，而不是图文宣传页

避免：

- 漂浮 blob 成为主视觉重点
- 过多英文标签破坏中文产品一致性
- 大面积空白导致首屏信息密度不足

#### 城市入口区

- 保留四城切换能力
- 视觉上从普通 tab 升级为“城市场景入口”
- 城市卡片要更像产品筛选，而不是单纯展示按钮

#### 核心能力区

需要更明确表达平台能力：

- 目的地推荐
- 地图导航
- 设施查询
- 旅游规划
- 日记交流

这些内容应被组织为产品能力入口或摘要卡，而不是零散内容堆叠。

#### 个性化区

- 已登录状态优先展示收藏、路线、计划等与当前用户相关的信息
- 未登录状态仍然有完整可理解的推荐内容，不出现空洞区域

### 3. 全局视觉 token 与组件风格

核心文件：`frontend/src/styles/main.css`

本次改版的关键不是逐页写新样式，而是重建全局视觉 token，让现有页面继承统一风格。

必须覆盖：

- 品牌色变量
- 页面背景与容器层级
- 导航链接
- 主按钮、次按钮、幽灵按钮
- 卡片容器
- 输入框、下拉框、文本域
- 状态标签、数字 pill、提示块
- 桌面与移动端顶部/侧边结构

实现原则：

- 尽量复用已有类名和结构，减少模板大改
- 能通过全局类收口的，不在单页写大量局部覆盖
- 让 `DestinationPage`、`RoutePage`、`PlanPage` 等页面在不重构布局的情况下获得一致升级

## 具体设计决策

### 品牌文案

- 产品主名统一为：`优途`
- 不再使用“城市漫游”作为品牌名
- “旅游日记”“旅游规划”等功能页标题可保留业务语义，但在品牌入口和首页表达上统一归属到“优途”

### 首页语气

- 中文为主，不强依赖英文副标题
- 文案重点是“能做什么”，不是抽象口号
- 每个入口模块都应体现工具价值，而不是营销口吻

### 交互密度

- 首页和壳子允许更精致，但功能页仍需保持工具型页面的扫描效率
- 不使用夸张 hero、过深层叠卡片或一屏只放一个 CTA 的 landing page 结构
- 地图导航相关页面继续优先清晰度，不被品牌视觉压过

## 验收标准

### 品牌一致性

- 全局品牌入口显示“优途”
- 侧边栏、移动端头部、首页首屏品牌表达一致
- 使用 `优途.png` 作为品牌标识来源

### 视觉一致性

- 全局主色从紫色体系切换为蓝橙体系
- 按钮、卡片、表单、导航、提示块形成统一视觉语言
- 首页与壳子在桌面端和移动端都能看出同一套产品风格

### 产品感

- 首页首屏能明确看出这是旅游/路线规划产品
- 主要功能入口清晰，不需要用户阅读长文案才能理解
- 页面不再有明显“课程模板”或“营销 landing”气质

### 兼容性

- 不影响现有路由和核心功能使用
- 不新增前端构建依赖
- `npm run typecheck`、`npm run lint`、`npm run build` 可通过；已有历史 warning 可维持，但不新增错误

## 实施建议

优先顺序：

1. 先改 `App.vue`，立住“优途”品牌和全局壳子
2. 再改 `main.css`，重设视觉 token 和核心组件样式
3. 最后改 `HomePage.vue`，让首页和新的品牌系统对齐

这样可以先统一骨架，再让首页成为新的展示入口，同时避免一开始就在多个页面上分散改动。

## 风险与边界

- 仓库当前存在未跟踪的 `.codex/` 和 `优途.png`，本次只将 `优途.png` 视为品牌资产，不默认提交 `.codex/`
- 当前前端文件中存在部分历史编码显示异常，本次改版应避免无关范围的大面积文案重写，优先控制在品牌入口与首页关键文案
- 若后续需要将 `RoutePage`、`DestinationPage`、`PlanPage` 做更明显的页面级重构，应在本次改版完成后单独出第二阶段设计
