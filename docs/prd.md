# PRD

本文档只保留产品范围。老师 PPT 硬性要求见 `course-requirements.md`，详细实现见 `architecture.md`、`api.md` 和 `algorithms.md`。

## 目标

建设一个面向北京高校与景区的个性化旅游系统，满足课程设计对真实数据、数据结构算法、前后端可运行系统和验收文档的要求。

## 用户

| 用户 | 主要需求 |
|---|---|
| 普通游客 | 查找目的地、规划路线、查询设施和美食。 |
| 校园游客 / 新生 | 在校园场景中完成室外路线和楼宇室内导航。 |
| 日记分享者 | 发布、浏览、检索、评分和 AI 辅助生成旅游日记内容。 |
| 数据维护人员 | 更新公开地理数据、初始化 SQLite、检查导入状态。 |

## 功能范围

- 目的地推荐、精选目的地和收藏：按热度、评价和兴趣排序，Top-10 不依赖完全排序。
- 搜索：按景点/学校名称、类别、关键字执行精确、前缀和多关键词查询。
- 路线：单目标最优路线、多目标闭环路线、地图展示、路径展示、距离/时间/拥挤度/交通方式策略。
- 室内导航：教学楼或景区建筑内部的大门、电梯、楼层、房间导航。
- 场所查询：按图距离查询附近超市、卫生间、图书馆、食堂等设施，并支持类别过滤。
- 美食推荐：按热度、评价、距离排序，支持菜系过滤和名称/菜系/饭店或窗口模糊查询。
- 日记：列表、详情、发布、浏览计数、评分、推荐、全文检索、压缩/解压。
- AI：日记草稿、AI 封面生成、根据照片和文字生成旅游动画分镜。
- 多用户：保存多个用户信息，登录后展示用户相关收藏、评分和互动状态。

## 非目标

- 不在 2c2g 部署环境运行数据抓取链路。
- 不把 PostgreSQL/PostGIS/Alembic 作为默认部署依赖。
- 不在生产路由注册 Admin/Agents 演示页面。
