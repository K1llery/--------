# PRD

本文档只保留产品范围，详细实现见 `architecture.md`、`api.md` 和 `algorithms.md`。

## 目标

建设一个面向北京高校与景区的个性化旅游系统，满足课程设计对真实数据、数据结构算法、前后端可运行系统和验收文档的要求。

## 用户

| 用户 | 主要需求 |
|---|---|
| 普通游客 | 查找目的地、规划路线、查询设施和美食。 |
| 校园游客 / 新生 | 在校园场景中完成室外路线和楼宇室内导航。 |
| 日记分享者 | 发布、浏览、检索、评分和 AI 辅助生成旅游日记内容。 |
| 数据维护人员 | 更新公开地理数据、初始化 SQLite、检查导入状态。 |

## 功能范围

- 目的地推荐、精选目的地和收藏。
- 精确搜索、前缀搜索和多关键词搜索。
- 单点路线、多点闭环路线和室内跨层路线。
- 附近设施、美食列表和真实图片兜底显示。
- 日记列表、详情、发布、浏览计数、评分、全文检索、压缩/解压。
- AI 日记草稿、AI 封面生成、日记 AIGC 分镜动画。

## 非目标

- 不在 2c2g 部署环境运行数据抓取链路。
- 不把 PostgreSQL/PostGIS/Alembic 作为默认部署依赖。
- 不在生产路由注册 Admin/Agents 演示页面。
