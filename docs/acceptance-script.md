# Acceptance Script

本文档保留课堂验收的主流程。老师 PPT 要求见 `course-requirements.md`，测试命令和覆盖范围见 `test-plan.md`。

| 顺序 | 演示内容 | 证据点 |
|---|---|---|
| 1 | 课程数据规模 | 展示目的地、建筑物/景点、设施类型、设施、边、用户数量满足 PPT 阈值。 |
| 2 | 登录和多用户 | `/api/auth/demo-accounts`、登录后用户状态、收藏或评分差异。 |
| 3 | 首页与精选目的地 | 真实高校/景区数据、精选目的地。 |
| 4 | 个性化推荐 | 热度、评分、兴趣排序；Top-K/Quickselect 推荐结果。 |
| 5 | 景点和学校搜索 | 名称、类别、关键字、前缀、多关键词检索结果。 |
| 6 | 指定地点路线 | POI 吸附到道路层后使用 Dijkstra/A*，展示最短距离、最短时间、拥挤度和步行/骑行/打车交通方式策略。 |
| 7 | 随便逛逛闭环路线 | 自动选择附近点位，复用 Held-Karp 或 Nearest Neighbor + 2-opt，从起点出发并返回起点。 |
| 8 | 最近设施路线 | 从图书馆查找最近公共厕所，单源最短路复用后的图距离排序，说明不是直线距离。 |
| 9 | 美食推荐 | 热度、评价、距离排序，菜系过滤和模糊查询。 |
| 10 | 室内导航 | 大门、电梯、楼层、房间和轮椅模式。 |
| 11 | 日记互动 | 列表、详情、浏览计数、登录评分、按热度/评分推荐。 |
| 12 | 日记检索与压缩 | 目的地/标题/全文检索，Huffman 压缩率和解压一致性。 |
| 13 | AI 能力 | 日记草稿、封面生图、AIGC 分镜播放预览。 |
| 14 | 文档和智能体 | 展示 `docs/`、`AGENTS.md`、`.github/agents/` 与课程 PPT 要求对应。 |

## 讲解重点

- 先说明生产数据走 SQLite，JSON 仅作为初始化和测试快照。
- 每个算法演示都对应 `algorithms.md` 中的职责和复杂度。
- 食物数据当前少于 10 条时，不要把“已能展示 10 个美食”作为验收表述；可表述为 Top-K 算法已支持，数据可继续扩充。
- 如果 AI Key 未配置，说明接口会返回明确业务错误，不影响核心验收链路。
