# Acceptance Script

本文档保留课堂验收的主流程。测试命令和覆盖范围见 `test-plan.md`。

| 顺序 | 演示内容 | 证据点 |
|---|---|---|
| 1 | 登录和演示账号 | `/api/auth/demo-accounts`、登录后用户状态。 |
| 2 | 首页与精选目的地 | 真实高校/景区数据、精选目的地。 |
| 3 | 个性化推荐 | Top-K/Quickselect 推荐结果。 |
| 4 | 搜索 | 精确、前缀、多关键词检索结果。 |
| 5 | 室外单点路线 | Dijkstra/A* 路线、距离和时间。 |
| 6 | 多点闭环路线 | Held-Karp 或 2-opt 路线顺序。 |
| 7 | 附近设施 | 单源最短路复用后的图距离排序。 |
| 8 | 美食推荐 | 场景和标签筛选。 |
| 9 | 室内导航 | 大门、电梯、楼层、房间和轮椅模式。 |
| 10 | 日记互动 | 列表、详情、浏览计数、登录评分。 |
| 11 | 日记压缩 | Huffman 压缩率和解压一致性。 |
| 12 | AI 能力 | 日记草稿、封面生图、AIGC 分镜播放预览。 |

## 讲解重点

- 先说明生产数据走 SQLite，JSON 仅作为初始化和测试快照。
- 每个算法演示都对应 `algorithms.md` 中的职责和复杂度。
- 如果 AI Key 未配置，说明接口会返回明确业务错误，不影响核心验收链路。
